# Спецификация текущего refinement-пакета layout

## Цель
Точечно улучшить routing и visual polishing без смены layout engine и без введения тяжёлого внешнего router-а.

## Scope
- уточнить port assignment для BPMN-шлюзов
- убрать прохождение стрелок через bounds самого шлюза, к оторому они подключаются
- исправить центровку `event/gateway` по центру shape, а не по footprint с label area
- уменьшить overlap у возвратных потоков
- исправить визуальное смещение подключения сверху/снизу к шлюзу в preview

## Правила для port assignment у шлюзов
Шлюз трактуется как один из трёх случаев:
- `split gateway`: один вход, несколько выходов
- `join gateway`: несколько входов, один выход
- `mixed gateway`: все остальные случаи

Правила:
- для `split gateway` вход фиксируется на левой стороне
- для `join gateway` выход фиксируется на правой стороне
- остальные стороны выбираются геометрически из доступных портов `top/right/bottom/left`
- повторное использование одной стороны допускается только если качественного уникального распределения нет

## Routing constraints
При выборе маршрута edge router должен учитывать:
- пересечения с чужими фигурами
- пересечения с уже проложенными линиями
- overlap collinear segments с уже проложенными линиями
- число изгибов
- длину маршрута

Дополнительное правило:
- после выхода из source-порта и до входа в target-порт линия не должна повторно заходить в bounds source/target node

## Центровка event/gateway
Post-layout alignment для `event/gateway` должен использовать:
- центр shape bounds
- а не footprint bounds, включающий label area

Если у элемента есть ровно один соседний `task` в том же lane, разрешается выровнять его по центру shape этого task.

## Подключение сверху/снизу к шлюзу
В preview верхняя и нижняя точки подключения должны вычисляться строго по центральной вертикальной оси shape шлюза.

Уточнение:
- опорная точка берётся из `shapeBounds`, а не из footprint/node container
- в preview путь должен сохранять явный прямой заход к порту, чтобы не возникало впечатления смещения вправо или влево
- BPMN DI export остаётся без специальных UI-скруглений

## Ожидаемый результат
- split/join шлюзы читаются ближе к BPMN-ожиданию
- стрелки не проходят через собственный шлюз
- `event/gateway` выровнены визуально лучше
- возвратные потоки меньше совпадают друг с другом по сегментам
- верхние/нижние подключения к шлюзам выглядят точно по центру фигуры
